## YEAR 100 — QA FOUNDATIONS

**Goal:** Produce graduates who are the guardians of product quality. Masters of manual testing, test automation, performance engineering, and the art of breaking software.

### Semester 1

**QA101 — Introduction to Software Quality Assurance** *(3 Credits)*
*Description:* Fundamentals of software testing — QA vs QC, SDLC, and how to identify, document, and track bugs.
*Classes:*
1. What is Quality Assurance? — QA vs QC, cost of bugs, testing principles
2. SDLC & STLC — Software Development vs Testing Life Cycle
3. Types of Testing — Functional vs Non-Functional, Black Box vs White Box
4. The Bug Life Cycle — New, Assigned, Open, Fixed, Retest, Verified, Closed
5. Test Scenarios vs Test Cases — What to test, how to write instructions
6. Writing Effective Test Cases — Pre-conditions, test data, expected results
7. Bug Reporting & Tools (Jira) — Clear bug reports, reproduction steps
8. Severity vs Priority — Business impact vs technical urgency
9. Functional Testing Techniques — Boundary Value Analysis, Equivalence Partitioning
10. User Acceptance Testing (UAT) — Alpha vs Beta, working with stakeholders
11. Introduction to API Testing — Testing endpoints with Postman
12. QA in Agile Teams — Role of QA in Scrum, shifting left

**PM103 — Introduction to Agile & Scrum** *(3 Credits)*
*Description:* Core Agile and Scrum principles for working in product teams.

**CS101 — Computer Science Fundamentals** *(3 Credits)*
*Description:* Core CS concepts — how software works, basic algorithms, and system fundamentals.

**DEV101 — JavaScript Fundamentals** *(3 Credits)*
*Description:* JavaScript basics needed for understanding web applications and writing automated tests.

**COM101 — Technical Communication & Bug Reporting** *(3 Credits)*
*Description:* Write clear, actionable bug reports and communicate effectively with developers.

### Semester 2

**QA102 — Test Planning & Documentation** *(3 Credits)*
*Description:* Create comprehensive test plans, test strategies, and test documentation for software projects.

**QA104 — Introduction to Test Automation (Selenium/Cypress)** *(3 Credits)*
*Description:* Automate browser tests with Selenium and Cypress — writing, running, and maintaining automated tests.

**DB101 — Database Fundamentals for Testers (SQL)** *(3 Credits)*
*Description:* SQL basics for testers — querying data, validating test results, and database testing.

**DEV102 — HTML & CSS Structure** *(3 Credits)*
*Description:* Understand web page structure for effective UI testing and element identification.

**QA106 — Mobile Application Testing** *(3 Credits)*
*Description:* Test mobile applications — iOS vs Android, emulators, real devices, and mobile-specific test cases.

## YEAR 200 — INTERMEDIATE QA

### Semester 1

**QA203 — Security Testing Fundamentals** *(3 Credits)*
*Description:* Identify security vulnerabilities — OWASP Top 10, penetration testing basics, and security scanning.

*Description:* Design scalable test automation frameworks — page object model, data-driven testing, and BDD.

**DEV202 — TypeScript Fundamentals** *(3 Credits)*
*Description:* TypeScript for test automation — types, interfaces, and TypeScript with testing frameworks.

### Semester 2

**QA205 — CI/CD Testing Integration** *(3 Credits)*
*Description:* Integrate automated tests into CI/CD pipelines — GitHub Actions, Jenkins, and test reporting.

**QA207 — Test Data Management** *(3 Credits)*
