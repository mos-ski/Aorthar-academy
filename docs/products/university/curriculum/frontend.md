*Quiz (5 questions):*
1. What is the difference between `let`, `const`, and `var`? — **Answer:** `let` is block-scoped and reassignable; `const` is block-scoped and not reassignable; `var` is function-scoped (older, less predictable)
2. What is a closure? — **Answer:** A function that remembers and has access to variables from its outer (enclosing) scope even after the outer function has finished executing
3. What is the difference between `==` and `===`? — **Answer:** `==` does type coercion before comparison; `===` checks both value and type (strict equality)
4. What is a Promise? — **Answer:** An object representing the eventual completion or failure of an asynchronous operation, with `.then()` and `.catch()` methods
5. What do `map()`, `filter()`, and `reduce()` do? — **Answer:** `map()` transforms each element and returns a new array; `filter()` returns elements that pass a test; `reduce()` accumulates all elements into a single value

2. What is the time complexity of binary search? — **Answer:** O(log n) — it halves the search space with each step

4. What is a hash table and what is its average lookup time? — **Answer:** A data structure that maps keys to values using a hash function; average lookup time is O(1)
5. What is recursion? — **Answer:** A function that calls itself with a base case to prevent infinite loops; useful for tree/graph traversal and divide-and-conquer algorithms

*Quiz (5 questions):*
1. What is binary and why do computers use it? — **Answer:** A base-2 number system using only 0s and 1s; computers use it because transistors have two states: on (1) and off (0)
2. What is the difference between RAM and storage? — **Answer:** RAM is fast, volatile memory for active processes; storage (SSD/HDD) is slower, persistent memory for long-term data
3. What happens when you type a URL in a browser? — **Answer:** DNS lookup → TCP connection → HTTP request → server processes → HTTP response → browser renders
4. What is an operating system? — **Answer:** Software that manages hardware resources and provides services for applications — it's the bridge between hardware and software
5. What is a process vs a thread? — **Answer:** A process is a running program with its own memory; a thread is a lightweight unit within a process that shares memory with other threads

1. What is JSX? — **Answer:** A syntax extension that lets you write HTML-like code in JavaScript; it gets compiled to React.createElement() calls
2. What is the difference between props and state? — **Answer:** Props are passed from parent to child (read-only); state is managed within a component and can change over time
3. What is the Virtual DOM? — **Answer:** A lightweight JavaScript representation of the real DOM; React uses it to batch updates and minimize expensive DOM operations
4. What is component composition? — **Answer:** Building complex UIs by combining smaller, reusable components rather than creating monolithic components
5. What does the key prop do in React lists? — **Answer:** It helps React identify which items have changed, been added, or removed — improving rendering performance

*Quiz (5 questions):*
1. What are the rules of hooks? — **Answer:** Only call hooks at the top level (not in loops/conditions); only call hooks from React functions (components or custom hooks)
2. When should you use useReducer instead of useState? — **Answer:** When state logic is complex, involves multiple sub-values, or when the next state depends on the previous state
3. What is a custom hook? — **Answer:** A JavaScript function that starts with "use" and calls other hooks; it lets you reuse stateful logic across components
4. What does useMemo do? — **Answer:** It memoizes an expensive computation, only recalculating when its dependencies change
5. What is the Context API used for? — **Answer:** Sharing state across the component tree without prop drilling — useful for themes, auth, locale, etc.

*Quiz (5 questions):*
1. What makes a good code review? — **Answer:** Constructive feedback, focus on architecture and clarity (not style), ask questions instead of demanding, and approve promptly when ready
2. What is the difference between a senior and staff engineer? — **Answer:** Senior engineers lead projects; staff engineers lead across teams, influence strategy, and mentor other engineers
