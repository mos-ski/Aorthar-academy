### **Course Deep Dive: QA101 - Introduction to Software Quality Assurance**

**Description**: This course covers the fundamentals of software testing. Students will learn the difference between QA and QC, the software development life cycle (SDLC), and how to identify, document, and track bugs effectively.

**Sample Quiz Questions (Weeks 1-4):**

1.  **What is the main difference between QA and QC?**
    a) QA is process-oriented; QC is product-oriented.
    b) QA is executing code; QC is planning.
    c) QA is done by developers; QC is done by testers.
    d) There is no difference.

2.  **Which phase of the STLC involves setting up the hardware and software to run tests?**
    a) Test Planning
    b) Test Environment Setup
    c) Test Closure
    d) Requirement Analysis

3.  **"White Box" testing primarily focuses on:**
    a) The user interface.
    b) Internal code structure and logic.
    c) User requirements.
    d) The installation process.

4.  **When a developer fixes a bug, what is the next status in the Bug Life Cycle?**
    a) Closed
    b) Verified
    c) Retest
    d) Deferred

**Final Exam Structure:**
- **Part A: Multiple Choice (40%)**: 40 questions covering testing terminology, processes, and methodologies.
- **Part B: Bug Report Writing (30%)**: Students are given a scenario (e.g., "The login button does nothing when clicked on mobile"). They must write a full bug report including Title, Severity, Priority, Steps to Reproduce, and Expected vs. Actual Result.
- **Part C: Test Case Design (30%)**: Students are given a simple feature requirement (e.g., "A password reset form"). They must write 5 distinct test cases (positive and negative) for this feature.
